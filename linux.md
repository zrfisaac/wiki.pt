# Linux

Linux is an open-source, Unix-like operating system kernel that serves as the foundation for a wide variety of operating systems (known as distributions or distros). Initially developed by Linus Torvalds in 1991, Linux has become one of the most widely used operating systems in the world, powering everything from personal computers and smartphones to servers, supercomputers, and embedded systems.

## Descrição de Cada Diretório no Linux

| Diretório   | Descrição                                                                 |
|-------------|---------------------------------------------------------------------------|
| `/`         | O diretório raiz, ponto de partida de toda a estrutura do sistema de arquivos. |
| `/bin`      | Contém binários essenciais do sistema, acessíveis a todos os usuários, como `ls` e `cp`. |
| `/boot`     | Contém arquivos de inicialização, incluindo o kernel e o carregador de boot (ex.: GRUB). |
| `/dev`      | Arquivos especiais que representam dispositivos de hardware, como discos (`sda`) e terminais (`tty`). |
| `/etc`      | Arquivos de configuração do sistema e scripts para inicialização e serviços. |
| `/home`     | Diretórios pessoais dos usuários, onde são armazenados seus arquivos e configurações. |
| `/lib`      | Bibliotecas compartilhadas essenciais para binários em `/bin` e `/sbin`. |
| `/media`    | Ponto de montagem para dispositivos removíveis, como pendrives e discos externos. |
| `/mnt`      | Diretório para montagem temporária de sistemas de arquivos. |
| `/opt`      | Software opcional instalado manualmente, geralmente aplicações de terceiros. |
| `/proc`     | Sistema de arquivos virtual que fornece informações sobre o kernel e processos em execução. |
| `/root`     | Diretório pessoal do superusuário `root`, separado dos outros usuários. |
| `/run`      | Armazena informações voláteis sobre o sistema, como PID de processos e sockets. |
| `/sbin`     | Binários essenciais para administração do sistema, como `fsck` e `iptables`. |
| `/tmp`      | Arquivos temporários criados por usuários ou processos, geralmente apagados em reinicializações. |
| `/usr`      | Contém programas e bibliotecas não essenciais para o sistema base, como ferramentas para usuários. |
| `/var`      | Armazena dados variáveis, incluindo logs do sistema, caches e filas de impressão. |
